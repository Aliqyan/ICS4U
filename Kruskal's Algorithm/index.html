Kruskal's Algorithm

Introduction

This Algorithm was created by Joseph Kruskal. Kruskal's Algorithm's finds the Minimum Spanning Tree(MST) of a graph, meaning that it finds the smallest total edge weight needed to conect all vertices of the graph. Thus the total number of edges in a MST will be equal to one less than the number of vertices in the tree. Kruskal's Algorithm is classified as a greedy Algorithm as it choses the smallest edge weights to create its MST.

Pseudo Code

KRUSKAL(G):
1 A = ∅
2 foreach v ∈ G.V:
3    MAKE-SET(v)
4 foreach (u, v) in G.E ordered by weight(u, v), increasing:
5    if FIND-SET(u) ≠ FIND-SET(v):
6       A = A ∪ {(u, v)}
7       UNION(u, v)
8 return A

Although this pseudo code seems complex, it is fairly easy to follow.

Long Explanation:

The above pseudo code innitializes an empty set A which will be the Minimum Spanning Tree. Line 2 and 3 are responsible for creating a set for every vertice, storing the vertices that are connected to it. Line 4 assumes that the edges are already in increasing order, (this can quickly be done using the Arrays.sort() method in java which utilises the quick sort algorithm), and then iterates through the edges. Whilst doing so it checks to see wether vertices u and v are in the same set. If they are that would mean adding the edge would create a cycle in the MST which is not allowed, so therefore the edge is ignored. Else, the edge (u,v) is added to the set A. the next step is to join the sets of u and v together as they are now connected. Lines 4 to 7 are then repeated until all the edges are covered. The MST is now created, and the pseudo code returns A. This Algorithm utilises the Union Find Algorithm which has been briefly covered here through the explanation of having vertice sets which are joined together.


Short Re-Cap:

  1. Sort the edges of the graph in increasing order
  2. Add the next sorted edge if it does not create a cycle
  3. Repeat step 2 untill all the vertices are connected
